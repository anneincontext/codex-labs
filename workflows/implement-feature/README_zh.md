# Implement Feature

[English](README.md) | **中文**

设计并交付一个有范围的 feature。

## 技能序列

```text
repo-understanding → architecture → (实现) → testing → code-review
```

| 步骤 | Skill | 目标 |
| ---- | ----- | ---- |
| 1 | [repo-understanding](../../skills/repo-understanding/) | 只动该动的模块 |
| 2 | [architecture](../../skills/architecture/) | feature 落点；API 形态 |
| 3 | _实现_ | Codex 改代码 + 你掌舵 |
| 4 | [testing](../../skills/testing/) | 新测试 + 原套件全绿 |
| 5 | [code-review](../../skills/code-review/) | 开 PR 前 review 完整 diff |

## 人工关卡

- 架构步骤后：你认可边界与对外 API
- PR 前：范围与最初需求一致（无顺手大重构）

## 完成标准

功能可用、主路径有测试、diff 已 review。
