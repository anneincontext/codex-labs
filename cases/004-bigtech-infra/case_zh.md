## 背景

**大厂 infra 工程师：** 刚从小公司跳进大厂，**入职不久，尚未上手业务代码**。在 infra 里，**工程师就是产品**——这是他对角色的理解，不是当前大厂工作的实录。

**公司给的 AI 额度：** 约 **$1000/月** 工具预算 + **Cursor Enterprise**（公司出，不是个人自费）。

### 小公司 vs 大公司（他的对比）

|                 | 小公司                                         | 大公司                                   |
| --------------- | ---------------------------------------------- | ---------------------------------------- |
| **技术债**      | 相对轻，改动成本低                             | **多** — 架构债、业务债叠在一起          |
| **工作流与 AI** | **更拥抱 AI** — 更愿意把软件生产当流水线来设计 | 组织惯性更大；业务交付往往压过工作流改造 |

他补充的一点（**不限于大厂**）：**harness 无论哪家公司都是一点点补的**，没有一步到位。

访谈时的整体印象：**不是工具不够**（大厂额度不低），而是大公司环境下 **技术债更重、节奏更快**，和小公司相比，**在工作流上拥抱 AI 没那么容易**——这是他刚进大厂、基于过往经验的判断，不是已经在大厂写代码后的总结。

### Harness / 流水线视角（小公司时期的经验）

以下主要来自**他在小公司**形成的看法，不是大厂现状：

- 软件生产像 **流水线** — 实践上的 **harness engineering**，但 **强领域相关**
- 测试平台、可观测性、中间件状态都重要
- 不能脱离 **业务领域** 谈 harness
- **写代码的 harness** 和 **工作流的 harness** 都重要

### 成熟度（他们的说法）

不是纯「升级换代」——**手写代码现在依然存在**，和后面几层并行：

1. 手写代码 — 仍在日常出现
2. Human-in-the-loop — harness 不够好；agent 从环境拿不到下一步信号
3. Multi-agent — **软件工程师定目标、设计流水线**

### 个人工具栈

个人在用、与当前大厂业务无直接关系：

- OpenClaw、**codex-cli**
- 国内 Claude 受限；共享 OpenAI 账号走中转
- Codex：直接聊或 SDK 自管——他说适合 **大仓库 / 代码解读**、面试回顾等

访谈里这一点后来有了落点：Anne 用同样方式读 [openai/codex](https://github.com/openai/codex)，**[codex-labs 这个 repo 就是这样诞生的](../005-learn-codex-repo/case_zh.md)**。他的体感没错 — **AI 在帮助人理解代码仓库这件事上确实很有效**，比硬啃源码起步成本低得多。
